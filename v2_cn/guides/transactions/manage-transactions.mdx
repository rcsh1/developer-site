本指南概述了取消、加速和放弃交易的过程。掌握这些技术可以确保您的交易得到及时准确的处理。

## 取消交易

取消交易是在交易仍处于待处理状态时停止它。

如果交易是错误发起的、需要修改其详细信息或不再需要执行，您可能希望取消交易。取消可以防止交易被进一步处理。

<Note>此操作仅适用于 MPC 钱包和智能合约钱包的交易。</Note>

如果交易的状态是以下之一，则可以取消：
- `Submitted`
- `PendingScreening`
- `PendingAuthorization`
- `PendingSignature`（仅当子状态为 `Queue`, `InsufficientBalance`, `InsufficientBalanceFundLocked`, `PendingSignerApproval`, `PendingSystemProcessing`, 或 `Built` 时）
    
要通过 WaaS 2.0 API 取消交易，请使用 [取消交易](/v2/api-references/transactions/cancel-transaction) 操作。这需要您希望取消的交易的交易 ID。

## Replace-By-Fee (RBF) 交易

Replace-By-Fee (RBF) 是一种允许您用新版本替换未确认交易的功能，新版本通常支付更高的交易费用。这种功能提供了在交易广播后但在确认之前修改交易的灵活性。RBF 的主要优点包括：

- 通过增加费用加快交易确认。
- 取消待处理交易。
- 修改交易详细信息，例如目的地址或转账金额。

RBF 的两个最常见用例是加速和放弃交易。

### 加速交易

加速交易利用 RBF 用更高费用的版本替换原始交易，鼓励矿工优先处理以更快确认。

如果交易因费用低而需要很长时间才能确认，您可能希望加速交易。这在网络拥堵时特别有用。

<Note>此操作仅适用于全托管钱包（Web3 钱包），MPC 钱包和智能合约钱包的交易。它不适用于以下链上的交易：VET、TRON、TVET、SOL 和 TON。</Note>

只有当交易状态为 `Broadcasting` 时，才能加速交易。

要加速交易，请使用 [加速交易](/v2/api-references/transactions/speed-up-transaction) 操作。这需要您希望加速的交易的交易 ID。

<Info>如果您加速智能合约钱包的交易，将触发两个 RBF 交易：一个用于智能合约钱包的交易，另一个用于委托人的交易。</Info>

### 放弃交易

放弃交易利用 RBF 用新版本替换原始交易，有效地取消它。
- 对于 EVM 链，RBF 交易的转账金额为 `0`，发送地址与接收地址相同。
- 对于 UTXO 链，RBF 交易的转账金额为 `0`，目的地址与原始交易中的找零地址相同。

如果交易是错误发起的、需要修改其详细信息或不再需要执行但无法取消，您可能希望放弃交易。

<Note>此操作仅适用于全托管钱包（Web3 钱包），MPC 钱包和智能合约钱包的交易。它不适用于以下链上的交易：VET、TRON、TVET、SOL 和 TON。</Note>

只有当交易状态为 `Broadcasting` 时，才能放弃交易。

要放弃交易，请使用 [放弃交易](/v2/api-references/transactions/drop-transaction) 操作。这需要您希望放弃的交易的交易 ID。

当交易被放弃时，任何后续的放弃或加速操作仍将应用于原始交易。例如，如果用户创建交易 A，然后对交易 A 执行放弃操作使用交易 B，接着对交易 B 执行加速操作使用交易 C，加速操作仍将应用于交易 A，而不是交易 B。

<Info>如果您放弃智能合约钱包的交易，将触发两个 RBF 交易：一个用于智能合约钱包的交易，另一个用于委托人的交易。</Info>

## Cobo 定义的交易类别

字段 `cobo_category` 由 Cobo 定义，用于表示交易的逻辑类别或用途，帮助用户识别交易属于哪类操作。

### Fee Station 相关

| 类别 | 描述 |
|---------|-------------|
| AutoFueling | Fee Station 代付 gas 费用的交易。 |
| AutoFuelingRefund | Fee Station 退回已代付 gas 费用的交易。 |
| BillPayment | Fee Station 为用户代付 Cobo Bills 的交易。 |
| BillRefund | Fee Station 退回已代付的 Cobo Bills 的交易。 |
| GasFeeSponsor | Fee Station 使用美元稳定币代付 gas 费用的交易。 |
| GasFeeSponsorRefund | Fee Station 退回使用美元稳定币代付的 gas 费用的交易。 |
| CommissionFeeCharge | Fee Station 代付一价全包费用的交易。 |
| CommissionFeeRefund | Fee Station 退回已代付的一价全包费用的交易。 |

<Note>Fee Station 相关交易及其类别，仅能通过 [Get Fee Station transaction information](/v2/api-references/feestation/get-fee-station-transaction-information) 和 [List all Fee Station transactions](/v2/api-references/feestation/list-all-fee-station-transactions) 获取。</Note>

### Screening 相关

| 类别 | 描述 |
|---------|-------------|
| ScreenAppRefund | 由 Screening App 发起的隔离交易退款。 |
| ScreenAppIsolate | 由 Screening App 发起的风险隔离交易。 |
| ComplianceRefund | 由 Cobo KYT 发起的合规隔离交易退款。 |
| ComplianceIsolate | 由 Cobo KYT 发起的合规隔离交易。 |

### Staking 相关

| 类别 | 描述 |
|---------|-------------|
| StakingBabylonBtcDelegation | 在 Babylon 质押中委托 BTC 的交易。 |
| StakingBabylonBtcSignBabylon | 在 Babylon BTC 质押流程中执行签名的交易。 |
| StakingBabylonClaim | 从 Babylon 质押中领取奖励的交易。 |
| StakingBabylonSlashing | Babylon 质押中与惩罚相关的交易。 |
| StakingBabylonStake | 在 Babylon 平台中质押资产的交易。 |
| StakingBabylonUnstake | 在 Babylon 平台中发起赎回的交易。 |
| StakingBabylonWithdraw | 从 Babylon 质押中提取资产的交易。 |
| StakingBithiveClaim | 从 BitHive 质押中领取奖励的交易。 |
| StakingBithiveStake | 在 BitHive 平台中质押资产的交易。 |
| StakingBithiveUnstake | 在 BitHive 平台中发起赎回的交易。 |
| StakingBithiveWithdraw | 从 BitHive 质押中提取资产的交易。 |
| StakingCoreStake | 在 Core 质押平台中质押资产的交易。 |
| StakingCoreWithdraw | 从 Core 质押平台中提取资产的交易。 |
| StakingEthClaim | 从 ETH 质押中领取奖励的交易。 |
| StakingEthStake | 质押 ETH 资产的交易。 |
| StakingEthUnstake | 赎回 ETH 资产的交易。 |
| StakingSkyfarmClaim | 从 Skyfarm 质押中领取奖励的交易。 |
| StakingSkyfarmStake | 在 Skyfarm 平台中质押资产的交易。 |
| StakingSkyfarmUnstake | 在 Skyfarm 平台中发起赎回的交易。 |

### 其他类别

| 类别 | 描述 |
|---------|-------------|
| AutoSweep | 自动将代币归集至统一地址的交易。 |
| Tokenization | 通过 Tokenization app 管理代币化资产（如 mint、wrap）的交易。 |
| TokenizationGasSponsor | 通过 Tokenization app 进行代币化操作时，由系统代付 gas 费用的交易，即使源地址余额不足也能继续执行。 |
| TokenSwap | 通过 Token Swap app 执行代币兑换或跨链桥接的交易。 |
| SafeTxMessage | 由 MPC 钱包发起的消息签名交易，用于授权智能合约钱包（Safe{Wallet}）的操作。 |
| StellarChangeTrust | 在 Stellar 网络上建立 trustline 的交易。 |
| NonceAccountSign | 使用 Solana nonce account 进行签名的交易，支持确定性且防重放的执行模式。 |
| FeePayerSign | 由 Solana 指定 fee payer 账户代付交易费用的交易，该账户与主签名账户相互独立。 |
